# TAXONOMY 1: inferred_stall_stage (Etapa donde ocurrió el estancamiento)
#   - SIMULATION_TERMS_DISCUSSION: Discusión/cuestionamiento de términos de oferta simulada
#   - HANDOFF_INITIATION: Recibió instrucciones pero no comenzó
#   - HANDOFF_DOCUMENT_UPLOAD: Comenzó recopilación de documentos pero abandonó
#   - HANDOFF_VALIDATION_WAIT: Envió documentos, se estancó esperando validación
#   - HANDOFF_REJECTED: Kuna comunicó rechazo después de validación
#   - GPS_SCHEDULING_STALL: Se estancó durante programación de cita GPS
#   - UNKNOWN_POST_SIMULATION: Punto de estancamiento post-simulación no claro

# TAXONOMY 2: primary_stall_reason_code (Razón del estancamiento)
#   - REJECTED_TERMS: Rechazo explícito de términos
#   - SILENCE_AFTER_TERMS: Dejó de responder, sin rechazo explícito
#   - CONFUSION_UNRESOLVED: Confusión sobre términos no resuelta
#   - FAILED_TO_INITIATE: Ignoró instrucciones de handoff
#   - ABANDONED_DURING_UPLOAD: Comenzó pero detuvo proceso a medio camino
#   - REQUIREMENTS_BARRIER: Expresó dificultad con requisitos específicos
#   - TECHNICAL_ISSUE: Mencionó errores técnicos con herramienta
#   - WAITING_VALIDATION: Abandonó durante espera prolongada post-envío
#   - REJECTED_BY_KUNA: Kuna comunicó rechazo
#   - GPS_SCHEDULING_FAILED: No pudo confirmar/rechazó cita GPS
#   - AGENT_DELAY: Desconexión tras demora significativa del agente
#   - EXPLICIT_DISENGAGEMENT: Manifestó pérdida de interés
#   - SILENT_DISENGAGEMENT: Silencio prolongado pese a seguimientos
#   - EXTERNAL_CIRCUMSTANCE: Mencionó razón personal externa
#   - PROCESS_COMPLETED: Lead aparentemente completó (problema de datos)
#   - AMBIGUOUS: Información insuficiente para determinar razón

# TAXONOMY 3: operational_insight_code (Área para mejora)
#   - CLARIFY_ELIGIBILITY: Mejorar comunicación inicial sobre requisitos
#   - IMPROVE_SIMULATION: Mejorar claridad/presentación de términos
#   - SIMPLIFY_HANDOFF: Facilitar inicio del proceso de handoff
#   - STREAMLINE_DOCS: Reducir complejidad/número de documentos
#   - ENHANCE_UX: Mejorar usabilidad/confiabilidad de herramientas
#   - OPTIMIZE_RESPONSE_TIME: Reducir demoras en respuestas
#   - REFINE_COMMUNICATION: Mejorar explicaciones y manejo de objeciones
#   - IMPROVE_AUTOMATION: Ajustar tiempo/claridad de mensajes automatizados
#   - REVIEW_VALIDATION: Examinar demoras en validaciones internas
#   - ENHANCE_GPS_SCHEDULING: Mejorar proceso de citas GPS
#   - ADDRESS_POLICY_CONCERN: Revisar/clarificar políticas de producto
#   - DATA_QUALITY_ISSUE: Posible error de datos o clasificación
#   - NO_OPERATIONAL_ISSUE: Estancamiento debido a factores externos

# IMPORTANT RULES:
- Output ONLY the YAML block - no other text before or after
- Follow field names and taxonomies exactly
- Make "next_action_detail" SPECIFIC and ACTIONABLE, starting with an imperative verb
- Base analysis SOLELY on the conversation text provided
